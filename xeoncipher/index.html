DATABASE — HOW IT WORKS & HOW YOU CAN HELP ⇨  IMPORTANT — READ BEFORE USING

EVERY ENTRY YOU FEED INTO THE HISTORY TABLE IS AUTOMATICALLY SUBMITTED TO THE BACKEND DATABASE — ITS TEXT AND ALL CIPHER VALUES ARE STORED AND BECOME SEARCHABLE FOR EVERY USER. YOU CAN ALSO QUERY THE DATABASE DIRECTLY: JUST TYPE ANY WORD OR PHRASE AND HIT THE DATABASE BUTTON WITHOUT ADDING IT TO YOUR HISTORY TABLE.

⚠  PLEASE KEEP THE DATABASE MEANINGFUL. AVOID SUBMITTING RANDOM STRINGS, TEST INPUTS, OR ENTRIES WITH NO RESEARCH VALUE — EVERY RECORD IS PERMANENT AND BAD DATA DEGRADES RESULTS FOR EVERYONE.

▸  THE DATABASE FEATURE MAY RETURN FEW OR NO MATCHES WHILE THE COLLECTION IS STILL GROWING. THIS IS EXPECTED. AS MORE USERS CONTRIBUTE QUALITY ENTRIES AND THE DATABASE IS PERIODICALLY EXPORTED TO THE GITHUB REPOSITORY, MATCH QUALITY WILL IMPROVE SIGNIFICANTLY. YOUR CONTRIBUTIONS NOW BUILD THE FOUNDATION FOR EVERYONE LATER.

⚠  THE DATABASE MAY ALSO RETURN NO RESULTS EVEN WHEN MATCHES EXIST — THIS IS A KNOWN LIMITATION OF THE CURRENT BACKEND DUE TO RESTRICTED READ/WRITE CAPACITY. IT IS NOT A BUG IN YOUR INPUT. THIS WILL BE RESOLVED ONCE THE DATABASE IS MIGRATED OFFLINE AND BUNDLED DIRECTLY INTO THE GITHUB REPOSITORY.
